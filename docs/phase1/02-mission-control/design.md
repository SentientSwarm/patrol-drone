## 1. Introduction
Two cooperating artifacts: `MissionStateMachine` (plain-Python, rclpy-free — owns every mission *decision*: arm, takeoff, waypoint-sequence-with-dwell, RTH, land, abort on low-battery/external-signal) and `patrol_mission` (thin rclpy node — owns every *mechanism*: sub `/fmu/out/*`, pub `/fmu/in/*`, single ENU→NED boundary, pub `/patrol/*`, fixed-rate timer driving `tick()`). Two launch files (`mission_basic.launch.py`, `mission_patrol.launch.py`) wire it; a versioned YAML makes the route reviewable. The decision/mechanism split is the precondition for the testing posture the PRD demands: all decision logic in a pure-Python class → every transition (incl. all abort paths) validated in a sub-5-second unit suite at ≥85% coverage with no ROS/Gazebo/PX4 started; the same `MissionStateMachine` carries unchanged into Phase 2 hardware bring-up. Owns (for 03/04/05 + Phase 2): mission/route YAML schema, `mission_*.launch.py`, `/patrol/{mission_state,current_waypoint,abort}`, the checkpoint-arrival capture-trigger semantic, the `MissionStateMachine` contract. Consumes: `/fmu/*` + `px4_msgs` (01), the shared checkpoint mapping (03), the bag-recorder launch include (05). Settled-but-unconfirmed consumed defaults (checkpoint mapping with 03; capture trigger with 04) are carried as Open Questions pending the combined cross-docset human review — not silently invented. (Source Projects + Related Projects tables: on-disk §1.)

## 2. Open Questions & Assumptions
All eight PRD OQs carried forward; six Resolved in design, two (OQ-2, OQ-7) **Resolved (combined review 2026-06-03)** for the combined cross-docset review.
- **OQ-1 — Resolved (hand-rolled state machine):** ~10 stable states; zero runtime dep on the Layer-A runner; import-clean/pickle-safe; one file to read (KISS). See §3.4, §4.2.3.
- **OQ-2 — Resolved (combined review 2026-06-03): checkpoint mapping schema AND file location.** Default: shared YAML at `sim/config/checkpoints.yaml` owned by 03, `{checkpoint_id, position{x,y,z} world/ENU, tag_family, tag_id}`. **Two parts pending:** (a) schema (field set) — matches 03 DoD §5, low-risk; (b) file location/name — 03 DoD §7 explicitly lists this as unresolved. Consuming code isolated from (b): the path is a single `MissionConfig` parameter, so a different agreed location is a one-line change. Joint with 03 §7 + 04.
- **OQ-3 — Resolved (`std_msgs`):** `/patrol/mission_state`=String, `/patrol/current_waypoint`=Int32, `/patrol/abort`=Bool(sub). Round-trip MCAP + render in Foxglove with no custom-type plugin; capture trigger reuses these (OQ-7) — no new message type owned here.
- **OQ-4 — Resolved:** `tolerance_m: 0.5`, `hold_time_s: 2.0` (plan illustratives as overridable YAML defaults; SITL-tunable without code change).
- **OQ-5 — Resolved (provisional budget):** one basic + one patrol (2-wp reduced) scenario, nightly SITL; ≤8 min/scenario incl. spin-up; quarantine-not-expand on >1-in-5 flake or >2× budget; never per-PR.
- **OQ-6 — Resolved:** `/fmu/out/battery_status.remaining` (0.0–1.0), default threshold `0.20` (YAML); transition unit-tested (AC-7); SITL-observable depletion best-effort.
- **OQ-7 — Resolved (combined review 2026-06-03): capture-trigger contract with 04.** Default: 02 emits observable `DWELL` state + index on `/patrol/current_waypoint` (maps 1:1 to `checkpoint_id` via mission YAML); 04 captures once per `DWELL` (satisfies 04 AC-6). No new topic/message owned by 02. Joint with 04.
- **OQ-8 — Resolved:** explicit home-waypoint offboard sequence behind the single `RTH` state (no offboard→RTL handoff; keeps control authority in the machine + unit-testable).
- **A-1 (assumption):** 01's `patrol_bringup`/`patrol_interfaces` shells exist + `colcon build` green before M1 — verified §3.5.
- **A-2 (assumption, INFERRED):** PX4 offboard needs a continuous setpoint stream (~10 Hz) before the mode switch — drives node keepalive; verified §3.5; node-plumbing only.

### 4.2 Component Architecture
**4.2.1 Inventory (6):** MissionStateMachine (pure Python, decisions), FrameConversion (pure Python, single ENU↔NED site), MissionConfig (pure Python, YAML loader+schema), PatrolMissionNode (rclpy, mechanism), Launch entry-points (config), Test suites — each with type/boundary/responsibility/dependencies (on-disk table).
**4.2.2 Dependency diagram (ASCII, on-disk):** launch→node→{config,frames,state_machine}; node↔px4_msgs/`/fmu/*`/PX4-SITL; pub `/patrol/*`; tests→pure-Python modules (unit) / launch→node→SITL (integration).
**4.2.3 MissionStateMachine** (`patrol_mission/state_machine.py`): states IDLE/ARMING/TAKEOFF/HOVER/WAYPOINT/DWELL/RTH/LANDING/ABORT/DONE (full entered/tick/exit table on-disk); frozen `Telemetry`/`Command` dataclasses; `tick(state, telem) -> (next_state, command)`; four abort guards (external, low-batt, manual-takeover[scaffold], timeout[scaffold]) evaluated first every tick (highest precedence), all routing ABORT→RTH; `_within_tolerance_for_hold(...)` records first-inside timestamp, completes only after continuous hold, never tests equality (MC-5); clock injected via `telem.now_s` for deterministic tests (INF-M1). *Traces to: MC-4/5/6/9/11 · UAC-MC-4/5/6/9 · INF-M1.*
**4.2.4 FrameConversion** (`patrol_mission/frames.py`): `to_ned_from_origin(point, frame, ekf_origin_ned)` — ned passthrough; enu→ned `(x,y,z)→(y,x,−z)` + origin offset; raise on unknown frame (fail loud). The single conversion site; a second site is review-rejectable (Tenet 4). *Traces to: MC-7 · UAC-MC-7.*
**4.2.5 MissionConfig** (`patrol_mission/config.py`): `Waypoint`/`AbortConfig`/`MissionConfig` frozen dataclasses; parse+validate mission YAML; resolve `checkpoint_id` against the checkpoint-positions YAML (03; **default path `sim/config/checkpoints.yaml` supplied as a launch arg / loader parameter so the OQ-2 file-location decision is a one-line config change, not a code edit**); no route data in source (MC-3); fail loud on missing field / unknown frame / unresolvable id (INF-M3). *Traces to: MC-3/5/6/7 · UAC-MC-3 · INF-M3.*
**4.2.6 PatrolMissionNode** (`patrol_mission/node.py`): 10 Hz timer — keepalive heartbeat, build Telemetry, `tick()`, translate Command→`/fmu/in/*`, pub `/patrol/*`; waypoint NED pre-resolved through FrameConversion at startup (EKF origin captured once at arm). Topic table: pub `/patrol/mission_state`(String, MC-8), pub `/patrol/current_waypoint`(Int32, MC-8), sub `/patrol/abort`(Bool latched, MC-6/8) + 6 `/fmu/*` px4_msgs topics. Arrival/capture trigger (OQ-7) is the observable `DWELL` + `current_waypoint`, NOT a new topic. *Traces to: MC-1/2/6/8 · UAC-MC-1/2/6/8 · INF-M2.*
**4.2.7 Launch entry-points** (`patrol_bringup/launch/{mission_basic,mission_patrol}.launch.py`): wiring only; `mission_patrol` also includes 05's recorder. *Traces to: MC-1/2/8 · UAC-MC-1/2.*
**4.2.8 Test suites** (`tests/unit/` ROS-free; `tests/integration/` SITL): Layer-A imports only pure-Python modules with mocked PX4 (<5 s, ≥85%); SITL drives `mission_*.launch.py` against real sim, nightly (don't mock the simulator). *Traces to: MC-4/9/10 · UAC-MC-4/9/10 · INF-M1.*
**4.2.9 Inventory Triangle Check:** inventory ↔ diagram ↔ consumer surface (`tick()` + `/patrol/*` + `mission_*.launch.py`) all enumerate the same six components; no drift.

### 4.3 Layer View
Config (MissionConfig, Launch), Decision-ROS-free (MissionStateMachine, FrameConversion), Plumbing (PatrolMissionNode), Flight-substrate (consumed PX4 via `/fmu/*`), Test. Notes: decision layer pure/deterministic/no-rclpy; plumbing follows px4_ros_com keepalive patterns — a node that branched on mission state is a review-rejectable layer violation.

### 4.4 Systemic / Platform Interfaces
**4.4.1 Summary:** Messaging (3 new `/patrol/*` std_msgs, P1), Observability (mission_state/current_waypoint ARE the observable surface, P1), Config (new mission YAML; consume checkpoints.yaml, P1), CI (unit suite + 85% gate; SITL nightly, P1), Security `[OOS]`.
**4.4.2 Messaging:** std_msgs so MCAP records + Foxglove renders with no plugin; QoS reliable/transient-local depth-1 (state/current), latched (abort); absent subscriber harmless; abort-never-published → low-battery abort still works.
**4.4.3 Security:** `[OOS: single-host pre-hardware sim — no network/users/tenancy/credentials; no PRD security FR]`. `/patrol/abort` deliberately unauthenticated in sim (empty threat model) and is the wiring point for an authenticated hardware trigger later (MC-11); no privilege boundaries to escalate across.
**4.4.4 CI / Test infra:** Layer-A coverage `source = patrol_mission/{state_machine,frames,config}.py`; tests import the three rclpy-free modules directly (resolves the ADR-0002 M3 coverage-source spike — Layer A needs no ROS toolchain); SITL flake never blocks PRs; quarantine-not-expand (Tenet 5).
**4.4.5 Cross-cutting Failure Modes (table on-disk):** stale/lost local_position (hold setpoint + keepalive, don't advance, log; resume or operator-abort); invalid YAML (fail loud at startup, refuse to fly); SITL/agent unreachable (stays in ARMING, never arms); offboard exit mid-flight (PX4 failsafe; surfaced on mission_state); low-battery crossing (intended ABORT→RTH, not a fault); 05 recorder absent (mission flies, no bag); identity/mesh `[OOS]`.
**4.4.6 Consumed cross-docset contracts (provisional):** checkpoint mapping (OQ-2 — schema + location, both pending) + capture trigger (OQ-7 — `DWELL`+index, 04 captures once per DWELL), both **confirmed at combined review (2026-06-03)**.

### 4.5 Key Interaction Sequences (ASCII diagrams on-disk)
Seq 1: basic arm→takeoff 5 m→hover 10 s→land (happy). Seq 2: patrol waypoint→tolerance+hold→DWELL (arrival signal→04)→next→RTH. Seq 3: RTH via explicit home-waypoint offboard sequence then land (OQ-8; RTL rejected — control authority + un-unit-testable). Seq 4: external-signal abort — guards evaluated first → ABORT→RTH, observable + recorded (AC-6); low-battery path identical, SITL-observable best-effort.

### 4.6 Data Model Changes
No DB. New checked-in YAML: `patrol_bringup/config/patrol_mission.yaml`, `…/mission_basic.yaml`. Consumed read-only: checkpoint-positions YAML (03, default `sim/config/checkpoints.yaml`, provisional OQ-2). New in-memory dataclasses: Telemetry/Command/Waypoint/AbortConfig/MissionConfig (runtime only).

### 4.7 UX Mocks
`[OOS: no graphical UI; operator interface is the CLI launch command; observable surface is `/patrol/*` rendered in Foxglove (owned by 05)]`. Operator state-surface table for `ros2 topic echo /patrol/mission_state` (IDLE→ARMING→TAKEOFF→HOVER→WAYPOINT→DWELL→RTH→LANDING→DONE, with current_waypoint index during the patrol) on-disk.

## 5. Design Questions FAQ
- **Q1 components/build order:** six components; node drives the state machine @10 Hz; build order state-machine+frames+config+unit (M1) → node+`mission_basic`+basic SITL (M1) → waypoint/dwell/abort/RTH+`mission_patrol`+patrol SITL (M2).
- **Q2 contracts:** `tick(current_state, telemetry) -> (next_state, command)`; 3 `/patrol/*` std_msgs + 6 `/fmu/*` px4_msgs (paths match §4.2.6 exactly); dataclasses; YAML schema Appendix C.2; no REST/DB.
- **Q3 infra:** runs in 01 containers/`ros2_ws`; new `patrol_mission` package; coverage `source` = three rclpy-free modules; SITL added to `sitl-nightly.yml`; no new runtime dep (hand-rolled, OQ-1); **PyYAML expected transitively via ROS 2 Jazzy base (`python3-yaml`), verified pre-M3 in §3.5, fallback `package.xml` `<exec_depend>`**.
- **Q4 external deps:** PX4/`px4_msgs`/01; checkpoints+world+camera/03; 05 recorder include (non-critical to flight) — each in §3.5 or §4.4.6.
- **Q5 testing:** unit Layer-A all transitions+abort(4)+frames+config <5 s ≥85% (PX4 mocked, sim NOT mocked); SITL nightly basic+patrol+external-abort — matches §6.2.
- **Q6 security:** OOS by design (§4.4.3), reasoned shorter than Q2; `/patrol/abort` unauthenticated intentionally; no escalation surface.
- **Q7 risks:** (1) offboard keepalive (A-2) — proven in M1 skeleton; (2) SITL flakiness (OQ-5) — small/strict/nightly/quarantine; (3) frame-conversion sprawl (MC-7) — single §4.2.4 boundary. Open: OQ-2 + OQ-7 remain Provisional and match §2 exactly (no Q7/§2 drift).

## 6. Implementation Plan
- **6.0 Linear:** Per-docset-local Linear project (this docset). Milestones **M1, M2, MZ** (MZ = terminal catch-all). Each milestone's Definition of Done includes a documentation true-up (§6.4); MZ accumulates deferred OQs/spikes, e2e/integration test expansion, and a final documentation+test consolidation (§6.5).
- **6.1 Milestone overview (walking-skeleton):** **M1 (plan M3, skeleton)** — basic mission node, demo `mission_basic.launch.py` arm→takeoff 5 m→hover 10 s→land (full stack crossed at minimum thickness; offboard-keepalive risk surfaces here). **M2 (plan M4, layer 1: patrol + safety + config + topics)** — `mission_patrol.launch.py` 4+ wp visit-order + dwell + RTH + land + mid-patrol abort. Each ships a demonstrable SITL flight. **MZ (terminal — consolidation & deferred backlog)** — no new flight capability; absorbs items surfaced during M1–M2 that shouldn't block them (see §6.5). Per-milestone Shippable-Demo + Exit-Criteria (AC tags) in the table on-disk.
- **6.2 Milestone details:** M1 tasks T1.1–T1.7 (basic states; frames; minimal config; unit suite; node+keepalive+`/fmu/*`; `mission_basic` launch; basic SITL test + coverage-source resolution). M2 tasks T2.1–T2.7 (WAYPOINT/DWELL/ABORT + all 4 abort guards; full YAML schema + checkpoint_id resolution; RTH offboard sequence; `/patrol/*` topics; abort/waypoint/RTH unit tests AC-7/8; `mission_patrol` launch + 05 include; patrol SITL test + external-abort assertion). Each milestone has an Out-of-Scope table (Source + Deferred-to), Files Touched on L/XL tasks, per-milestone unit + SITL testing tables, and documentation artifacts (all on-disk).
- **6.3 Layered delivery:** M1 = thinnest full-stack flight (offboard-keepalive risk surfaces here, not late); M2 = thicken to patrol + safety + downstream contracts. Scope-shedding: shed M2, M1 alone shippable; abort paths are a hard floor (unit-tested regardless of SITL observability). Parallel: T1.2/T1.3 independent of T1.1.
- **6.4 Definition of Done (every milestone):** tasks code-reviewed; unit (≥85%, <5 s) + milestone SITL pass; shippable demo runs end-to-end; **documentation trued up — PRD/Design/DoD/README reconciled against what was actually built this milestone**; no P1 bugs; §4.4 systemic interfaces integrated.
- **6.5 MZ (terminal milestone):** catch-all for work that surfaces during M1–M2 but isn't blocking. Seeded with: **OQ-5** SITL-runtime budget re-measure (replace the provisional ≤8 min/scenario with a measured figure); e2e/integration test-suite expansion beyond the two canonical scenarios; a comprehensive documentation + test consolidation pass; tech-debt/hardening logged during the build. **Exit:** MZ reviewed and either cleared or items explicitly punted to Phase 2 — "project done" requires this.
