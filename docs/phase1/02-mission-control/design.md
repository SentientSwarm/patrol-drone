## 1. Introduction
Two cooperating artifacts: `MissionStateMachine` (plain-Python, rclpy-free ‚Äî owns every mission *decision*: arm, takeoff, waypoint-sequence-with-dwell, RTH, land, abort on low-battery/external-signal) and `patrol_mission` (thin rclpy node ‚Äî owns every *mechanism*: sub `/fmu/out/*`, pub `/fmu/in/*`, single ENU‚ÜíNED boundary, pub `/patrol/*`, fixed-rate timer driving `tick()`). Two launch files (`mission_basic.launch.py`, `mission_patrol.launch.py`) wire it; a versioned YAML makes the route reviewable. The decision/mechanism split is the precondition for the testing posture the PRD demands: all decision logic in a pure-Python class ‚Üí every transition (incl. all abort paths) validated in a sub-5-second unit suite at ‚â•85% coverage with no ROS/Gazebo/PX4 started; the same `MissionStateMachine` carries unchanged into Phase 2 hardware bring-up. Owns (for 03/04/05 + Phase 2): mission/route YAML schema, `mission_*.launch.py`, `/patrol/{mission_state,current_waypoint,abort}`, the checkpoint-arrival capture-trigger semantic, the `MissionStateMachine` contract. Consumes: `/fmu/*` + `px4_msgs` (01), the shared checkpoint mapping (03), the bag-recorder launch include (05). Settled-but-unconfirmed consumed defaults (checkpoint mapping with 03; capture trigger with 04) are carried as Open Questions pending the combined cross-docset human review ‚Äî not silently invented. (Source Projects + Related Projects tables: on-disk ¬ß1.)

### 4.2 Component Architecture
**4.2.1 Inventory (6):** MissionStateMachine (pure Python, decisions), FrameConversion (pure Python, single ENU‚ÜîNED site), MissionConfig (pure Python, YAML loader+schema), PatrolMissionNode (rclpy, mechanism), Launch entry-points (config), Test suites ‚Äî each with type/boundary/responsibility/dependencies (on-disk table).
**4.2.2 Dependency diagram (ASCII, on-disk):** launch‚Üínode‚Üí{config,frames,state_machine}; node‚Üîpx4_msgs/`/fmu/*`/PX4-SITL; pub `/patrol/*`; tests‚Üípure-Python modules (unit) / launch‚Üínode‚ÜíSITL (integration).
**4.2.3 MissionStateMachine** (`patrol_mission/state_machine.py`): states IDLE/ARMING/TAKEOFF/HOVER/WAYPOINT/DWELL/RTH/LANDING/ABORT/DONE (full entered/tick/exit table on-disk); frozen `Telemetry`/`Command` dataclasses; `tick(state, telem) -> (next_state, command)`; four abort guards (external, low-batt, manual-takeover[scaffold], timeout[scaffold]) evaluated first every tick (highest precedence), all routing ABORT‚ÜíRTH; `_within_tolerance_for_hold(...)` records first-inside timestamp, completes only after continuous hold, never tests equality (MC-5); clock injected via `telem.now_s` for deterministic tests (INF-M1). *Traces to: MC-4/5/6/9/11 ¬∑ UAC-MC-4/5/6/9 ¬∑ INF-M1.*
**4.2.4 FrameConversion** (`patrol_mission/frames.py`): `to_ned_from_origin(point, frame, ekf_origin_ned)` ‚Äî ned passthrough; enu‚Üíned `(x,y,z)‚Üí(y,x,‚àíz)` + origin offset; raise on unknown frame (fail loud). The single conversion site; a second site is review-rejectable (Tenet 4). *Traces to: MC-7 ¬∑ UAC-MC-7.*
**4.2.5 MissionConfig** (`patrol_mission/config.py`): `Waypoint`/`AbortConfig`/`MissionConfig` frozen dataclasses; parse+validate mission YAML; resolve `checkpoint_id` against the checkpoint-positions YAML (03; **default path `sim/config/checkpoints.yaml` supplied as a launch arg / loader parameter so the OQ-2 file-location decision is a one-line config change, not a code edit**); no route data in source (MC-3); fail loud on missing field / unknown frame / unresolvable id (INF-M3). *Traces to: MC-3/5/6/7 ¬∑ UAC-MC-3 ¬∑ INF-M3.*
**4.2.6 PatrolMissionNode** (`patrol_mission/node.py`): 10 Hz timer ‚Äî keepalive heartbeat, build Telemetry, `tick()`, translate Command‚Üí`/fmu/in/*`, pub `/patrol/*`; waypoint NED pre-resolved through FrameConversion at startup (EKF origin captured once at arm). Topic table: pub `/patrol/mission_state`(String, MC-8), pub `/patrol/current_waypoint`(Int32, MC-8), sub `/patrol/abort`(Bool latched, MC-6/8) + 6 `/fmu/*` px4_msgs topics. Arrival/capture trigger (OQ-7) is the observable `DWELL` + `current_waypoint`, NOT a new topic. *Traces to: MC-1/2/6/8 ¬∑ UAC-MC-1/2/6/8 ¬∑ INF-M2.*
**4.2.7 Launch entry-points** (`patrol_bringup/launch/{mission_basic,mission_patrol}.launch.py`): wiring only; `mission_patrol` also includes 05's recorder. *Traces to: MC-1/2/8 ¬∑ UAC-MC-1/2.*
**4.2.8 Test suites** (`tests/unit/` ROS-free; `tests/integration/` SITL): Layer-A imports only pure-Python modules with mocked PX4 (<5 s, ‚â•85%); SITL drives `mission_*.launch.py` against real sim, nightly (don't mock the simulator). *Traces to: MC-4/9/10 ¬∑ UAC-MC-4/9/10 ¬∑ INF-M1.*
**4.2.9 Inventory Triangle Check:** inventory ‚Üî diagram ‚Üî consumer surface (`tick()` + `/patrol/*` + `mission_*.launch.py`) all enumerate the same six components; no drift.

### 4.3 Layer View
Config (MissionConfig, Launch), Decision-ROS-free (MissionStateMachine, FrameConversion), Plumbing (PatrolMissionNode), Flight-substrate (consumed PX4 via `/fmu/*`), Test. Notes: decision layer pure/deterministic/no-rclpy; plumbing follows px4_ros_com keepalive patterns ‚Äî a node that branched on mission state is a review-rejectable layer violation.

### 4.4 Systemic / Platform Interfaces
**4.4.1 Summary:** Messaging (3 new `/patrol/*` std_msgs, P1), Observability (mission_state/current_waypoint ARE the observable surface, P1), Config (new mission YAML; consume checkpoints.yaml, P1), CI (unit suite + 85% gate; SITL nightly, P1), Security `[OOS]`.
**4.4.2 Messaging:** std_msgs so MCAP records + Foxglove renders with no plugin; QoS reliable/transient-local depth-1 (state/current), latched (abort); absent subscriber harmless; abort-never-published ‚Üí low-battery abort still works.
**4.4.3 Security:** `[OOS: single-host pre-hardware sim ‚Äî no network/users/tenancy/credentials; no PRD security FR]`. `/patrol/abort` deliberately unauthenticated in sim (empty threat model) and is the wiring point for an authenticated hardware trigger later (MC-11); no privilege boundaries to escalate across.
**4.4.4 CI / Test infra:** Layer-A coverage `source = patrol_mission/{state_machine,frames,config}.py`; tests import the three rclpy-free modules directly (resolves the ADR-0002 M3 coverage-source spike ‚Äî Layer A needs no ROS toolchain); SITL flake never blocks PRs; quarantine-not-expand (Tenet 5).
**4.4.5 Cross-cutting Failure Modes (table on-disk):** stale/lost local_position (hold setpoint + keepalive, don't advance, log; resume or operator-abort); invalid YAML (fail loud at startup, refuse to fly); SITL/agent unreachable (stays in ARMING, never arms); offboard exit mid-flight (PX4 failsafe; surfaced on mission_state); low-battery crossing (intended ABORT‚ÜíRTH, not a fault); 05 recorder absent (mission flies, no bag); identity/mesh `[OOS]`.
**4.4.6 Consumed cross-docset contracts (provisional):** checkpoint mapping (OQ-2 ‚Äî schema + location, both pending) + capture trigger (OQ-7 ‚Äî `DWELL`+index, 04 captures once per DWELL), both **confirmed at combined review (2026-06-03)**.

### 4.5 Key Interaction Sequences (ASCII diagrams on-disk)
