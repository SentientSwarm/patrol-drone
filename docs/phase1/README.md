# Phase 1 docsets

Phase 1 of the autonomous-drone plan, decomposed into five PRD/Design docsets for the
prd-engine `/drive` lifecycle. The source of truth for *what* Phase 1 delivers is
[`../phase1_simulation_plan.md`](../phase1_simulation_plan.md); each docset below begins
with a formal Definition-of-Done (`dod.md`) and gains `prd.md` + `design.md` as `/drive` runs.

## Phase 1 docset traceability matrix

**Build / dependency order:** `01-platform → 02-mission-control → 03-sim-environment → 04-perception → 05-logging-replay` (mirrors milestone order M1→M8). Each docset stands on the ones before it; 01 is the foundation (depends on nothing internal), 05 is the terminal consumer.

**Integrative exit-checklist items** (no single solo owner — assembled across docsets): **#1** (full end-to-end patrol run — primary 02, depends on 01/03/04/05) and **#10** (README setup-to-running-mission in <20 commands — primary 01, spans all docsets).
